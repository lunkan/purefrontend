PureFrontend

Powerful front-end script for fast prototyping and clean front-end deliveries. No backend needed - import, repeat and replace elements by pure front-end markup.

Download PureFrontend

What is PureFrontend

PureFrontend use the html comment element to import or repeat markup in order to make templates clean and easy to update. Importent content may contain PureFrontend markup as well. PureFrontend will execute all actions from bottom up - A child element repeat/import will be executed before it's parents.

We can also add dummy text by adding a range of "text-variables" to the markup. These variables will be replaced with Lorum ipsum content depending on the variable name. You can find all avalible variables here. Below is an example of PureFrontend markup (Last character is replaced with "*" to prevent the variable from being executed).

    
						
  1. ...
    2.

    4. 
						
    5. 
						
  5. <div class="container"></span>

    8. 
						
  8.   <div class="text-block">
    9. 
						
  9.     <h2>{ titl* }</h2>
    10. 
						
  10.     <!-- repeat(3) -->
    11.

Install PureFrontend

Download PureFrontend and include the js-script at the bottom of the html page. If your page consists of more than one page you must add the js-script at the bottom of all navigable pages. Content imported by the PureFrontend "import-feature" does not need the js-script.

    
							
  1.     ...
    2. 
							
  2.     <script src="purefrontend.js" type="text/javascript"></script>
    3. 
							
  3.   </body>
    4. 
							
  4. </html>
    5.

PureFrontend on local file system

The main reason to use PureFrontend is that you don't have to bother about servers - any directory in your filesystem will do. For security reason you may find your browser blocking files when imported from the local filesystem. There is different workaround to bypass this restriction depending on what browser you are using.

Chrome

In Chrome you can access local files by adding the project as an extension. Add a manifest file to the directory where your filse to be imported are located. You may copy the code below and replace the files in "web_accessible_resources" with your own file-names. Make sure you save the file as a JSON-file "manifest.json".

    
							
  1. {
    2. 
							
  2.   "name": "File holder",
    3. 
							
  3.   "version": "1.0",
    4. 
							
  4.   "manifest_version": 2,
    5. 
							
  5.   "web_accessible_resources": ["head.html", "footer.html", "header.html"]
    6. 
							
  6. }
    7.

After the manifest-file is created - load the extension: Visit chrome://extensions -> check "Developer Mode" and click "Load unpacked extension" to activate it. You will now have your extension visible in the list with an extension-id attached to it. Add the extension-id to your html-templates as described below. Done!

    
							
  1.     ...
    2. 
							
  2.     <script>
    3. 
							
  3.       var extensionId = "gjghekipjcljkianfcmhhamhdcndbjde";
    4. 
							
  4.     </script>
    5. 
							
  5.     <script src="purefrontend.js" type="text/javascript"></script>
    6. 
							
  6.   </body>
    7. 
							
  7. </html>
